Как попасть в мир ИИ: от браузера до своего сервера
БЕСПЛАТНЫЕ СПОСОБЫ

🌐 Браузер и смартфон

  • Чат-интерфейсы: Гигачат, Алиса, DeepSeek, Qwen, Kimi
  • Не требует установки, работает сразу
  • Ограничения: функционал, лимиты, зависимость от интернета

💻 Ollama на ноутбуке

  • Запуск открытых моделей локально
  • Полная приватность, нет зависимости
  • Ограничения: нужны ресурсы (ОЗУ/GPU), только открытые модели
ОПЛАТА ЧЕРЕЗ API

☁️ Облачный API провайдер

  • Платите за токены: вход/выход
  • Примеры: российские и международные платформы
  • Плюсы: нет администрирования, масштабируется
  • Минусы: данные во внешнем облаке

🖥️ Аренда GPU облачный сервер

  • Платите за время работы видеокарты
  • Примеры: SberCloud, Yandex Cloud, Selectel, Timeweb
  • Плюсы: контроль над моделью и данными
  • Минусы: платите за простой, нужны навыки развёртывания
СВОЙ СЕРВЕР ЛОКАЛЬНО
  • Покупка железа + эксплуатация
  • Плюсы: максимальный контроль, безопасность, низкая себестоимость при высокой нагрузке
  • Минусы: высокие стартовые вложения, нужна инфраструктура и администрирование

Регулярная нагрузка + контроль → аренда GPU
Высокая нагрузка + строгая безопасность → свой сервер

ТРИ УРОВНЯ ДАННЫХ

Публичные

  • Новости, открытые статьи, публичные данные
  • Можно отправлять в любые чаты

Внутренние но не критичные

  • Черновики, идеи, неперсональные рабочие заметки
  • Только в чаты с политикой «данные не используются для обучения»

Конфиденциальные персональные

  • Персданные (152-ФЗ), договоры, финансы, пароли, код, стратегии
  • Только локальное развёртывание (Ollama, свой сервер) или российские облака
ЧТО ПРОИСХОДИТ С ВАШИМИ ДАННЫМИ
  • Бесплатные чаты: часто используют диалоги для дообучения моделей
  • Платные тарифы: обычно исключают данные из обучения (читайте Terms!)
  • Локально: данные никуда не уходят
Правило: Если не уверены — не отправляйте. Или анонимизируйте.
🇷 ОСОБЕННОСТИ В РФ
  • 152-ФЗ: персональные данные должны обрабатываться на территории РФ
  • Санкции: доступ к некоторым моделям только через посредников или VPN
  • Российские платформы: Гигачат, Алиса, SberCloud — соответствуют требованиям, но функционал может отличаться
Практический совет: Создайте внутренний чек-лист:
✓ Это публичные данные → можно в любой чат
✓ Это внутренние но не персданные → только чаты с политикой «не обучать»
✓ Это конфиденциально/персданные → только локально или российское облако с соглашением
Экономика внедрения ИИ
Сколько стоит использовать ИИ

💰 МОДЕЛЬ 1: Оплата за токены API

  • Вход: ~$0.1-0.5 за 1 млн токенов
  • Выход: ~$0.2-2.0 за 1 млн токенов
  • Пример: 1000 запросов/день × 2000 токенов = ~$10-50/месяц
  • Плюсы: Нет вложений, масштабируется
  • Минусы: Данные во внешнем облаке, цена растёт
Для пилотов, нерегулярной нагрузки

💰 МОДЕЛЬ 2: Аренда GPU (облако)

  • RTX 4090: ~₽50-150/час
  • A100/H100: ~₽300-800/час
  • Российские платформы: SberCloud, Yandex Cloud, Selectel
  • Плюсы: Контроль над моделью и данными
  • Минусы: Платите за простой, нужны навыки
Для регулярной нагрузки >500 запросов/день, требования к конфиденциальности

💰 МОДЕЛЬ 3: Своё железо (CapEx)

  • Железо: RTX 4090 ~₽150-200 тыс., сервер 4×GPU ~₽1.5-3 млн
  • В год: электричество ₽30-100 тыс., администрирование 0.2-1.0 FTE
  • Плюсы: Максимальный контроль, безопасность
  • Минусы: Высокие вложения, инфраструктура
Для высокой нагрузки >2000 запросов/день, строгие требования к локализации

Простая формула выбора

Меньше 100 запросов/день → Облачный API (токены)
100-1000 запросов/день → Аренда GPU или гибридное решение
Больше 1000 запросов/день → Своё железо (считайте окупаемость)
Правило: Если проект живёт дольше 1 года и нагрузка стабильна — своё железо чаще выгоднее аренды
Не забудьте скрытые расходы:
Плюс 30-50% на безопасность | Плюс 30-50% на мониторинг и обновления

От простого поиска к автономным действиям
🔍

УРОВЕНЬ 1: Поиск в интернете

«Найди актуальное»
ИИ обращается к веб-источникам в реальном времени
Для новостей, курсов, событий, публичных данных
Когда нужен свежий ответ, а не «знания на дату обучения»
Не заменяет ваши внутренние данные!

📚

УРОВЕНЬ 2: RAG (Retrieval-Augmented Generation)

«Найди в МОИХ документах»
Вы загружаете свои файлы → ИИ ищет ответы ТОЛЬКО в них, а не «придумывает»
Для корпоративных инструкций, договоров, переписки, личных заметок
Борьба с галлюцинациями: «Не знаешь — скажи "не знаю"»
Лайфхак для любого чата: загрузить файл + строгая инструкция «Ищи ТОЛЬКО здесь»

🔬

УРОВЕНЬ 3: Исследования

«Найди → проанализируй → уточни → сделай вывод»
Многошаговый поиск с ПЛАНИРОВАНИЕМ и АДАПТАЦИЕЙ
Это первая АГЕНТСКАЯ возможность!
ИИ сам решает: что искать дальше, когда информации «достаточно», как структурировать ответ
Пример: «Подбери тур: бюджет, даты, требования — найди, сравни, порекомендуй»

🤖

УРОВЕНЬ 4: Полноценные агенты

«Спланируй → используй инструменты → сделай»
ИИ не только ищет информацию, но и ВЫПОЛНЯЕТ действия через ВНЕШНИЕ ИНСТРУМЕНТЫ
Инструментом может быть ЛЮБАЯ программа: поиск, код, почта, Jira, Git, API, даже программа, написанная самим агентом «на лету»!
Из чего состоит агент: Планировщик → Диспетчер инструментов → Память → Критик
⚠️ Важно: Агент сам принимает решения о последовательности действий. «Доверяй, но проверяй!»

⚠️ ТИПИЧНЫЕ СЦЕНАРИИ СБОЕВ

🔄 Бесконечный цикл

ИИ ищет лучшую цену → поиск не завершается → новый статус → исчерпан лимит токенов

🎯 Неправильная интерпретация цели

«Оптимизируй расходы» → агент отменил все подписки, включая критичные

🛠️ Ошибка в инструменте

Агент написал код с багом → запустил → повредил данные / отправил не туда

🤝 Конфликт агентов

Два агента пытаются изменить один документ → потеря данных / противоречия

Лимиты на время выполнения и число шагов. Песочница для исполнения кода. Запрет на доступ к критичным системам

Чек-лист: с чего начать
Конкретные шаги для внедрения ИИ
1

ШАГ 1: Определите задачу

  • Что именно должен делать ИИ
  • Какой результат считается успешным
  • Какие данные нужны для этого
2

ШАГ 2: Оцените требования

  • Безопасность и 152-ФЗ
  • Объём данных: сколько запросов в день
  • Бюджет на внедрение и эксплуатацию
3

ШАГ 3: Выберите способ доступа

  • Бесплатный чат для тестов
  • Ollama для приватных экспериментов
  • Облачный API для пилота
  • Аренда на своём сервере для продакшена
4

ШАГ 4: Подготовьте данные

  • Очистите и структурируйте
  • Анонимизируйте персональные данные
  • Подготовьте примеры для few-shot промптов
5

ШАГ 5: Настройте промпты

  • Используйте шаблоны из лекции
  • Протестируйте на реальных запросах
  • Задокументируйте рабочие промпты
6

ШАГ 6: Запустите пилот

  • Ограничьте охват приложения
  • Наблюдайте за качеством ответов
  • Собирайте обратную связь
7

ШАГ 7: Масштабируйте

  • Оптимизируйте процессы
  • Автоматизируйте рутинные задачи
  • Расширяйте функционал постепенно

Ключевые метрики успеха

❌ Качество ответов, удовлетворённость пользователей ⏱️ Скорость выполнения задачи 💰 Стоимость одного запроса 🔧 Время на поддержку и доработку
